# 通讯行业核心数据类型梳理笔记

> 基于通讯行业常见分类，梳理数据类型的大类/小类、产生方式、含义功能及在大数据采集清洗后的运用场景。

---

## 一、数据类型总览（大类与小类）

| 大类 | 小类 | 简要说明 |
|------|------|----------|
| **网络管理与性能数据** | PM（性能管理） | 设备/网元 KPI，量化网络质量与容量 |
| | CM（配置管理） | 设备参数、邻区、频点等配置与状态 |
| | FM（故障管理） | 告警、故障事件，用于定位与恢复 |
| | 4/5G 无线网管数据 | 无线侧 PM/CM/FM 的集合，来自基站等网管 |
| **DPI 深度解析数据** | 4/5G-DPI | 移动网业务识别、用户级流量与质量 |
| | VoLTE-DPI | 语音业务信令与媒体解析、语音质量评估 |
| | 物联网-DPI | 物联网终端连接、流量与业务质量 |
| | 固网-DPI | 宽带侧业务识别与体验评估 |
| **运营与业务数据** | B 域（业务支撑） | 用户、套餐、账单、营销等 |
| | M 域（管理支撑） | 财务、人力、采购等内部管理 |
| | 共建共享网络数据 | 多运营商共建 5G 的协同配置与指标 |

## 二、网络管理与性能数据

### 2.1 大类说明

来源于**网管系统**（OSS），面向网络运维与优化，是“网络运行状态”的数字化表达。

### 2.2 小类：PM（性能管理）

| 项目 | 说明 |
|------|------|
| **含义与功能** | 通过采集网元/设备的 **KPI**（如覆盖率、掉线率、接入成功率、时延、吞吐量），量化网络运行质量，评估容量与用户体验，支撑规划与优化。 |
| **如何产生** | 网元（基站、核心网网元等）按周期（如 15 分钟、1 小时）统计计数器或指标，通过北向接口（如 FTP、SFTP、接口机）上报给网管；网管再提供给采集系统或大数据平台。 |
| **典型内容** | 无线：RSRP/RSRQ、掉线率、切换成功率、PRB 利用率等；核心网：附着成功率、寻呼成功率等。 |
| **采集清洗后运用** | 指标监控大屏、容量与拥塞分析、覆盖/质量专题分析、根因分析（与 DPI/工参关联）、预测性维护、报表与考核。 |

### 2.3 小类：CM（配置管理）

| 项目 | 说明 |
|------|------|
| **含义与功能** | 管理网络设备的**参数与状态配置**（如基站频点、邻区关系、功率、PCI 等），保证配置一致、变更可追溯，是网络稳定与优化策略落地的“基石”。 |
| **如何产生** | 来自网管或配置中心的**配置库/配置快照**；通过北向接口或人工导出，按版本/时间点提供；部分也来自工参表（如经数据仓库加工的“工参”）。 |
| **典型内容** | 小区/基站基础参数、邻区关系、频率与带宽、天线参数、PCI 等。 |
| **采集清洗后运用** | 工参库、邻区与覆盖仿真、参数一致性核查、变更影响分析、与 PM/DPI 关联做“小区级”分析、地理化呈现。 |

### 2.4 小类：FM（故障管理）

| 项目 | 说明 |
|------|------|
| **含义与功能** | **实时监控与识别**网络中的故障与告警，帮助运维快速定位、派单与恢复，缩短故障时长，减少对业务和用户的影响。 |
| **如何产生** | 网元或网管在检测到门限越限、通信中断、设备异常等事件时产生**告警/故障事件**，通过北向接口或消息推送至网管、运维平台或大数据平台。 |
| **典型内容** | 告警级别、告警类型、网元/网元类型、发生/恢复时间、告警文本等。 |
| **采集清洗后运用** | 告警监控、故障关联与根因、影响范围分析（结合 CM/工参）、SLA 与 MTTR 统计、与 PM/用户投诉关联分析。 |

### 2.5 小类：4/5G 无线网管数据

| 项目 | 说明 |
|------|------|
| **含义与功能** | 特指从 **4G/5G 无线接入网**（基站、gNB/eNB 等）网管系统采集的、用于无线侧运维与优化的数据，是 PM、CM、FM 在**无线侧**的具体体现。 |
| **如何产生** | 无线网管从基站及控制器采集性能计数器、配置快照、告警事件，经北向接口或文件接口提供给上层 OSS 或大数据采集。 |
| **典型内容** | 无线 PM（如 4G/5G 的 KPI）、无线 CM（小区/邻区等）、无线 FM（基站/小区级告警）。 |
| **采集清洗后运用** | 无线专题分析、覆盖与干扰分析、容量规划、与 DPI/工参联合做用户感知分析、共建共享场景下的协同优化。 |

## 三、DPI 深度解析数据

### 3.1 大类说明

通过对**网络侧数据包**进行深度检测与解析，识别**具体业务与应用**（如 APP、协议），提供**用户级/业务级**的流量、行为与质量数据，与网管数据互补。

### 3.2 小类：4/5G-DPI 数据

| 项目 | 说明 |
|------|------|
| **含义与功能** | 对 4G/5G **核心网接口**（如 S1-U、N3 等）的数据包做 DPI，识别业务与应用（如抖音、微信），提供**用户级**的业务使用行为、流量构成与质量感知。 |
| **如何产生** | 在核心网侧部署 DPI 探针或采集点，对用户面/控制面报文进行解析与关联，产出 XDR/会话级流水（用户、时间、业务、流量、时延等）。 |
| **典型内容** | 用户标识、小区/位置、业务/应用类型、上下行流量、时延、成功率等。 |
| **采集清洗后运用** | 业务与流量报表、用户行为分析、套餐与流量经营、与 PM/工参关联做“小区×业务”质量分析、精准营销与体验优化。 |

### 3.3 小类：VoLTE-DPI 数据

| 项目 | 说明 |
|------|------|
| **含义与功能** | 针对 **VoLTE 语音业务** 的 DPI 数据，通过解析信令与媒体包，**端到端评估语音质量**，精准定位“听不清”“打不通”等问题的原因（如无线、核心网、对端）。 |
| **如何产生** | VoLTE 相关接口（如 SBC、CSCF、媒体面）部署 DPI，解析 SIP/RTP 等，生成 VoLTE 话单或质量事件（接通率、时延、MOS、丢包等）。 |
| **典型内容** | 主被叫、起呼/接续/释放时间、接通/未接通原因、媒体质量指标（MOS、丢包、抖动）等。 |
| **采集清洗后运用** | 语音质量监控、未接通与质差根因分析、与无线 PM/工参关联定位弱覆盖或干扰、投诉预处理与 SLA 分析。 |

### 3.4 小类：物联网-DPI 数据

| 项目 | 说明 |
|------|------|
| **含义与功能** | 面向 **物联网业务**（如车联网、智能表计、工业网关）的 DPI 数据，监控物联网终端的连接状态、**流量模型**与业务质量，保障行业应用可靠性。 |
| **如何产生** | 在物联网专用 APN 或核心网出口部署 DPI，识别物联网协议与业务，按终端/卡/业务产出连接与流量流水。 |
| **典型内容** | 终端/卡标识、连接/断开、流量、业务类型、质量事件等。 |
| **采集清洗后运用** | 物联网连接与流量报表、异常终端与流量检测、行业大客户保障、故障与质量定界。 |

### 3.5 小类：固网-DPI 数据

| 项目 | 说明 |
|------|------|
| **含义与功能** | 在 **固定宽带网络**（如 OLT、城域网）部署 DPI，洞察家庭或企业宽带用户的**互联网业务体验**，实现分用户、分业务的网络质量可视化管理。 |
| **如何产生** | 在宽带接入或汇聚侧部署 DPI 探针，解析用户上网流量，识别业务与应用，产出用户/会话级流水。 |
| **典型内容** | 用户/端口、业务类型、流量、时延、卡顿等体验指标。 |
| **采集清洗后运用** | 家宽/政企业务质量报表、体验与投诉分析、与 B 域结合做套餐与提速推荐、大网质量评估。 |

## 四、运营与业务数据

### 4.1 大类说明

来源于 **BSS/ERP 等业务与管理系统**，描述“用户与产品”“企业内部运营”，与网络数据结合可做经营分析、精准营销与成本管控。

### 4.2 小类：B 域（业务支撑域）

| 项目 | 说明 |
|------|------|
| **含义与功能** | **业务支撑域**：包含用户基本信息、消费习惯、套餐、账单、订单、客服记录等，用于市场营销、客户关系管理、套餐与收入分析。 |
| **如何产生** | 由 CRM、计费、订单、客服等 BSS 系统在业务办理、计费出账、投诉处理等流程中产生并入库；通过数据同步、接口或数仓采集。 |
| **典型内容** | 用户 ID、套餐、ARPU、账单、终端、渠道、投诉工单等。 |
| **采集清洗后运用** | 用户画像、套餐与流量经营、流失预警、精准营销、与 DPI/网管数据关联做“用户×网络”体验与价值分析。 |

### 4.3 小类：M 域（管理支撑域）

| 项目 | 说明 |
|------|------|
| **含义与功能** | **管理支撑域**：包含财务、人力、采购、投资等**企业内部管理**数据，用于企业运营决策、成本精算与资源调配。 |
| **如何产生** | 由 ERP、财务、人力、供应链等系统在日常经营管理中产生；通过数据仓库或数据湖接入。 |
| **典型内容** | 成本、收入、投资、人力、采购、项目等。 |
| **采集清洗后运用** | 成本与收入分析、投资效益评估、与 B 域/网络数据结合做单用户成本、单站效益等分析。 |

### 4.4 小类：共建共享网络数据

| 项目 | 说明 |
|------|------|
| **含义与功能** | 在**多运营商共建共享 5G 网络**（如电信与联通）模式下，用于**协同运营**的数据，包括统一参数配置、性能指标、故障信息等，保证双方用户感知一致与网络高效协同。 |
| **如何产生** | 由共建共享管理平台或双方网管对接，汇总两侧的 CM/PM/FM 及部分 DPI 数据，经标准化与脱敏后共享。 |
| **典型内容** | 统一工参、统一 KPI、故障与告警、资源与承载划分等。 |
| **采集清洗后运用** | 共建共享大屏与报表、双方 KPI 对标、故障协同处理、容量与扩容协同、与各自 B 域结合做用户感知与价值分析。 |

## 五、大数据采集与清洗后的典型运用场景汇总

| 数据类型 | 采集与清洗要点 | 典型运用场景 |
|----------|----------------|----------------|
| **PM** | 周期对齐、指标标准化、异常值处理 | 质量大屏、容量与拥塞分析、根因分析、预测维护 |
| **CM** | 版本与时间对齐、与工参主数据融合 | 工参库、邻区与覆盖分析、参数核查、地理化与关联分析 |
| **FM** | 告警归一、去重、与网元/时间关联 | 故障监控、根因与影响分析、SLA/MTTR、与 PM/投诉关联 |
| **4/5G 无线网管** | 与统一工参/地理信息关联 | 无线专题、覆盖/干扰/容量、与 DPI 做感知分析 |
| **4/5G-DPI** | 用户与小区关联、业务标签统一 | 业务与流量报表、用户行为、套餐经营、小区×业务质量 |
| **VoLTE-DPI** | 信令与媒体关联、质量指标标准化 | 语音质量、未接通/质差根因、投诉与 SLA |
| **物联网-DPI** | 终端/卡与行业映射 | 物联网报表、异常检测、行业保障与定界 |
| **固网-DPI** | 用户/端口与 B 域关联 | 家宽/政企体验、投诉分析、套餐与提速推荐 |
| **B 域** | 主数据治理、与网络侧 ID 打通 | 用户画像、营销、流失预警、价值与体验联合分析 |
| **M 域** | 与项目/网元/用户维度对齐 | 成本与效益、投资评估、单站/单用户成本 |
| **共建共享** | 双方口径与维度统一 | 协同运维、KPI 对标、容量与故障协同 |

## 六、小结：三类数据的关系与协同

- **网络管理与性能数据（PM/CM/FM）**：描述“网络好不好、怎么配、哪里坏”，是运维与优化的基础。
- **DPI 数据**：描述“谁、在哪儿、用了什么业务、体验如何”，把网络与**业务、用户**连接起来。
- **B/M 域数据**：描述“用户是谁、花了多少钱、公司成本与收入”，与网络和 DPI 结合可做**价值与体验**一体化分析。

在数仓或大数据平台中，通常以 **ODS 接入 → DWD 清洗与关联（如统一工参、用户 ID）→ DWS/ADS 按场景汇总** 的方式，将上述类型组合使用，支撑运维、优化、经营与决策等场景。
