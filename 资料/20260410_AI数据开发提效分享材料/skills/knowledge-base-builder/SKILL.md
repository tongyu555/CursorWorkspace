# 知识库构建技能

## 触发条件
当用户要求进行以下操作时触发：
- 构建知识库 / 整理知识库
- 分析 Hive 资产 / 索引 DDL
- 为新项目创建数据字典
- 整理表结构 / ETL 索引 / 数据血缘

## 前置准备

### 1. 确认项目信息
向用户确认以下信息：
- **项目名称**：用于创建项目子目录和命名知识库文件
- **DDL 文件位置**：包含 CREATE TABLE 语句的 SQL 文件路径
- **ETL SQL 代码位置**：生产环境 SQL 代码的目录路径
- **（可选）模型文档**：Excel/文档形式的业务模型说明

### 2. 创建项目目录
在 `D:\000_zhaoxc\01_zhaoxc\04_project\000_doc\<项目名>\` 下创建标准目录结构：
```
<项目名>/
├── DDL/          -- DDL源文件
├── sql_code/     -- ETL SQL代码
├── 知识库_表资产目录.json
├── 知识库_ETL任务索引.json
├── 知识库_数据血缘.json
├── 知识库_数据资产全景.md
├── 知识库_表结构速查手册.md
└── 知识库构建提示词.md
```

## 执行流程

### 第一步：DDL 解析
1. 扫描 DDL 目录下所有 `.sql` 文件
2. 使用正则表达式提取 CREATE TABLE 语句
3. 对每张表提取：
   - 表名、列定义（字段名、类型、注释）
   - 分区字段（PARTITIONED BY）
   - 存储格式（STORED AS / ROW FORMAT SERDE）
   - 表属性（TBLPROPERTIES）
   - Location
4. 根据表名自动推断：
   - **数仓层级**：表名前缀（dm_=DM, aggr_=AGGR, ods_=ODS, fact_=FACT, dw_=DW, cfg_=CFG, dim_=DIM）
   - **时间粒度**：表名后缀（_d=天, _h=小时, _w=周, _m=月, _q=季度, _y=年）
   - **业务域**：表名中的关键词（lte=4G, nr=5G, volte=语音, dpi=深度包检测等）

### 第二步：ETL SQL 扫描
1. 扫描 sql_code 目录下所有 `.sql` 文件
2. 对每个文件提取：
   - 目标表：从 INSERT INTO/OVERWRITE TABLE 语句提取
   - 来源表：从 FROM/JOIN 子句提取
   - 任务组：根据文件所在目录名确定
3. 统计任务组分布

### 第三步：数据血缘构建
1. 基于 ETL 扫描结果，建立表级血缘关系
2. 格式：`{"produced_by": {"目标表": ["来源表列表"]}}`

### 第四步：输出知识库文件
输出 5 个标准文件（格式定义参见 `D:\000_zhaoxc\01_zhaoxc\04_project\000_doc\通用知识库整理提示词.md`）：
1. `知识库_表资产目录.json` - 结构化表元数据
2. `知识库_ETL任务索引.json` - ETL任务索引
3. `知识库_数据血缘.json` - 表级血缘
4. `知识库_数据资产全景.md` - 总览文档
5. `知识库_表结构速查手册.md` - 字段级速查

### 第五步：创建项目规则
在 `.cursor/rules/` 下创建 `1x-<项目名>-project.mdc`（编号：10=DITO, 11=下一个项目...），内容包括：
- 项目标识和数据库前缀
- DDL校验路径
- 知识库引用路径
- 分区命名惯例
- 数仓分层和业务域说明

### 第六步：质量验证
- 表资产目录中的表数量 = DDL解析数量
- ETL任务数 = sql_code文件数
- 血缘关系中的所有表名存在于表资产目录中
- 全景文档统计与JSON数据一致

## 更新 README
完成后更新 `D:\000_zhaoxc\01_zhaoxc\04_project\000_doc\README.md`，在项目索引中添加新项目的信息。

## 大规模数据处理
- DDL 超过 10 万行时，使用 Python 脚本辅助解析
- SQL 文件超过 1000 个时，分批扫描（每批 200-500 个）
- JSON 输出文件超过 10MB 时，考虑按业务域拆分

## 输出确认
完成后向用户报告：
1. 构建统计摘要（表数、任务数、血缘关系数）
2. 识别的核心业务表 TOP 10
3. 总结的 3 条最重要的查询规范
4. 知识库文件清单和存放路径
