# SQL 审查技能

## 触发条件
当用户要求对 SQL 文件进行以下操作时触发：
- 审查 / review / 检查 / 质检
- 格式优化 / 规范检查
- 代码评审

## 执行流程

### 1. 加载审查规则
读取 `.cursor/rules/02-sql-review-rules.mdc`，加载36条审查规则（R001-R036）。

### 2. 读取目标SQL
读取用户指定的 SQL 文件（单个或批量）。

### 3. 逐规则检查
按优先级顺序对SQL代码进行检查：

**P0 必须检查（高优先级）：**
- R003：是否存在 SELECT *
- R004：多表JOIN字段是否有表别名前缀
- R008：特殊字段名是否用反引号包围
- R009：类型转换是否使用 CAST
- R010：算术运算是否处理空值
- R014：是否使用显式 JOIN 语法
- R017：分区表是否带分区条件
- R023：语法完整性（括号、引号、结束符）
- R025：字段名有效性
- R030：动态分区是否超限

**P1 强烈建议：**
- R001/R002：表别名命名和层级一致性
- R006/R007：字段对齐和逗号风格
- R012：运算符空格
- R015/R016：JOIN类型和子查询优化
- R024：字段重复选择
- R027：作者和时间信息
- R032：DISTINCT使用合理性

**P2 建议检查：**
- R013：SET语句语法
- R020/R021/R022：小文件和shuffle配置
- R029：Hive配置检查

### 4. 识别项目上下文
- 检查文件路径或内容中是否包含项目标识（如DITO）
- 如果是DITO项目，额外加载 `.cursor/rules/10-dito-project.mdc` 中的项目专属规则
- 根据项目知识库验证表名和字段名

### 5. 输出审查报告
按 `.cursor/rules/04-review-report-format.mdc` 中定义的格式输出报告：
- 执行摘要（健康度评级、一句话评价、关键风险）
- 基本信息（任务名、作者、输入输出表、复杂度）
- 语法规范审查（问题列表，标注行号、代码片段、涉及规则）
- 性能风险审查
- 数据质量风险审查
- 审查指标评分（规范性/性能/稳定性/可维护性/数据质量，0-5分）
- 检查规则覆盖情况

报告输出为 Markdown 文件，文件名与输入SQL文件名一致，存放在同目录下。

## 批量审查模式
当用户指定目录或多个文件时：
1. 逐文件执行审查流程
2. 在目录下输出每个文件的独立报告
3. 额外输出一份汇总报告，包含所有文件的评分和共性问题

## 注意事项
- 审查过程中不修改原始SQL文件
- 对于无法确认的问题（如字段名有效性），标注"需人工确认"
- 报告中引用代码片段时保留原始行号
