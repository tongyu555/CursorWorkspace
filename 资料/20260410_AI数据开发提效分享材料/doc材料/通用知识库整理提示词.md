# 通用知识库整理提示词模板

> 本模板用于为新项目构建 Hive 数据资产知识库。复制本文档并替换 `<项目名>` 为实际项目名称即可使用。

---

## 使用方法

1. 在 `D:\000_zhaoxc\01_zhaoxc\04_project\000_doc\` 下创建 `<项目名>/` 目录
2. 将项目的 DDL 文件放入 `<项目名>/DDL/` 目录
3. 将项目的 ETL SQL 代码放入 `<项目名>/sql_code/` 目录
4. 将下方提示词粘贴到 Cursor 对话中，引导 AI 构建知识库

---

## 提示词正文

```
你现在是一名深度理解业务逻辑的大数据架构师。你的任务是分析并索引 D:\000_zhaoxc\01_zhaoxc\04_project\000_doc\<项目名> 目录中的所有 Hive 相关资源，构建一个用于辅助生成取数 SQL 的知识库。构建的知识库也存放于该目录下。

## 输入资源

该目录包含：
1. `DDL/` 文件夹：所有 Hive 表的结构定义（CREATE TABLE 语句）
2. `sql_code/` 文件夹：生产环境的 ETL SQL 代码，代表了最权威的业务口径
3. （可选）`model/` 文件夹：模型设计文档

## 构建任务

请按以下步骤构建知识体系：

### 第一步：DDL 解析
- 扫描 `DDL/` 目录下的所有 .sql 文件
- 提取每张表的：表名、列定义（字段名、类型、注释）、分区字段、存储格式、Location
- 根据表名前缀自动推断数仓层级（DM/AGGR/ODS/FACT/DW/CFG/DIM等）
- 根据表名后缀推断时间粒度（_d=天/_h=小时/_w=周/_m=月/_q=季度/_y=年）
- 根据表名关键词推断业务域

### 第二步：ETL SQL 扫描
- 扫描 `sql_code/` 目录下的所有 .sql 文件
- 提取每个 ETL 任务的：文件路径、目标表（INSERT INTO/OVERWRITE）、来源表（FROM/JOIN）
- 按目录分组为任务组
- 提取关键逻辑：过滤条件、聚合方式、JOIN 关系

### 第三步：数据血缘构建
- 基于 ETL SQL 的解析结果，建立表之间的生产-消费关系
- 格式：{"produced_by": {"目标表": ["来源表1", "来源表2", ...]}}

### 第四步：知识库文件输出

请输出以下 5 个标准文件：

1. **知识库_表资产目录.json**
   ```json
   {
     "summary": {
       "total_tables": <数量>,
       "layer_distribution": {"DM": <n>, "AGGR": <n>, ...},
       "granularity_distribution": {"天级": <n>, "小时级": <n>, ...},
       "business_domain_distribution": {"域名": <n>, ...}
     },
     "tables": {
       "<表名>": {
         "layer": "<层级>",
         "granularity": "<粒度>",
         "business_domains": ["<域>"],
         "columns": [{"name": "<列名>", "type": "<类型>", "comment": "<注释>"}],
         "partition_keys": [{"name": "<分区列>", "type": "<类型>"}],
         "storage_format": "<格式>",
         "location": "<路径>"
       }
     }
   }
```

1. **知识库_ETL任务索引.json**
  ```json
   {
     "summary": {"total_tasks": <数量>, "task_group_distribution": {"<组名>": <n>}},
     "tasks": {
       "<文件路径>": {
         "target_table": "<目标表>",
         "source_tables": ["<来源表>"],
         "task_group": "<任务组>"
       }
     }
   }
  ```
2. **知识库_数据血缘.json**
  ```json
   {
     "produced_by": {"<目标表>": ["<来源表1>", "<来源表2>"]}
   }
  ```
3. **知识库_数据资产全景.md**
  包含：资产总览（分层/粒度/业务域统计）、数仓分层架构图、业务域分类索引、分区规范、高频表速查、数据血缘概览、取数SQL编写指南、ETL任务组索引
4. **知识库_表结构速查手册.md**
  按层级逐表列出：表名、业务域、粒度、分区字段、完整字段表（字段名/类型/注释）

### 第五步：质量检查

完成后请验证：

- 表资产目录中的表数量与DDL解析数量一致
- ETL任务数量与sql_code文件数量一致
- 血缘关系中的表名在表资产目录中都存在
- 全景文档的统计数据与JSON一致
- 速查手册覆盖了所有表

## 输出要求

分析完成后，请简要告诉我：

1. 你已经理解了哪些核心业务表
2. 你总结出的 3 条最重要的查询规范
3. 知识库构建的统计摘要（表数、任务数、血缘关系数）

```
