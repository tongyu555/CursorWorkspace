# AI 构建数据资产知识库 — 分享材料使用说明

本文档说明 `20260410` 分享包内各目录与文件的用途，便于大家按需查阅或在自己的 Cursor 工程中复现类似能力。

---

## 一、整体结构（先看这里）

| 目录 / 文件 | 用途简述 |
|-------------|----------|
| **doc材料/** | 知识库核心文档与 DITO 示例数据（DDL、JSON/Markdown 知识库、提示词、归档与开发规则）。**建议从本目录的 `README.md` 与下文「doc材料」小节开始读。** |
| **skills/** | 可直接拷贝到个人项目 `.cursor/skills/` 下的 **Cursor Skill**（告诉 AI 何时、如何帮你做知识库构建、SQL 审查、临时取数、问题排查）。 |
| **cursor对话样例/** | 若干 **脱敏/示范** 的对话记录，展示典型问法与 AI 协作方式（知识库构建、临时取数、历史经验取数、问题排查）。 |
| **excel格式转换脚本/** | 将模型 Excel 转为 TOON/JSON 等便于 AI 读取格式的 **脚本与工具**（含可执行程序时需自行评估运行环境）。 |
| **使用说明.md** | 本文件：说明分享包每一项是干什么的。 |

---

## 二、`doc材料/` 目录

### 2.1 入口与索引

- **`README.md`**：数据资产知识库的 **总索引**，说明标准目录结构、DITO 项目技能对应关系、如何扩展新项目等。分享包中的路径若与你们本地工程不一致，以实际放置位置为准，对照结构理解即可。

### 2.2 通用与 DITO 专用提示词

- **`通用知识库整理提示词.md`**：面向 **任意新项目** 的通用模板。复制后替换项目名、目录路径，在 Cursor 里引导 AI 从 DDL + ETL SQL 等资产生成表目录、血缘、索引等知识库产物。
- **`DITO/知识库构建提示词.md`**：针对 **DITO** 已落地知识库的「加载上下文」类提示词，用于让 AI 先读已有 JSON/Markdown 产物并遵守分区与口径规范。

### 2.3 DITO 知识库主体（`DITO/`）

以下为已生成或可扫描的 **知识库产物与源数据**（体积较大的 JSON 为表级/任务级索引，适合给 AI 或程序检索，不必人工通读）。

| 文件 / 子目录 | 用途 |
|----------------|------|
| **`知识库_表资产目录.json`** | 表级元数据（表名、分层、业务域、分区等），取数/建模时的「表清单」入口。 |
| **`知识库_ETL任务索引.json`** | ETL 任务与脚本侧索引，便于对齐任务名与产出表。 |
| **`知识库_数据血缘.json`** | 表级血缘，用于追溯上游/下游依赖。 |
| **`知识库_数据资产全景.md`** | 资产总览类文档，适合人读。 |
| **`知识库_表结构速查手册.md`** | 字段级速查，适合快速查字段含义与关联。 |
| **`DDL/exported-sql-ddl_*.sql`** | Hive DDL 源文件，**生成/校验 SQL 时以实际表结构为准**。 |
| **`DDL/` 下部分 `*_scan.json` / `.md`**、**`DITO/_region_tables_scan_result.json`** 等 | 针对区域/地理等主题的 **扫描或分析中间结果**，用于复盘「如何从 DDL 里做主题归纳」；一般使用者可跳过，需要深挖主题时再打开。 |
| **`DITO/_scan_region_tables.py`** | 与上述扫描相关的 **辅助脚本示例**（需本地 Python 环境与路径配置）。 |

#### 关于本分享包中的「空目录 / 占位说明」

为控制体积，部分与全量工程一致的目录在分享包中 **仅保留文件夹占位，未附带全部文件**：

| 占位目录 | 说明 |
|----------|------|
| **`DITO/sql_code/`** | 对应生产侧 **ETL SQL 任务组**（全量工程中有数百个任务组）。若需「口径最权威」的加工逻辑，请在有权限的环境从正式代码库或数仓文档同步至此目录后再让 AI 扫描。 |
| **`DITO/模型_toon/`** | **TOON 格式模型文件** 全量体积大，分享包内可能为空；需要时可从正式工程拷贝或使用本包 `excel格式转换脚本` 自行从 Excel 生成。 |
| **`DITO/模型_excel/`** | 可能仅含示例或临时文件；完整 SDM/分层建模 Excel 请以团队资产库为准。 |

> **使用建议**：有 DDL + 已生成的 JSON/Markdown 知识库，已能支撑大部分取数与问答；补齐 `sql_code/` 后，AI 对齐业务口径的能力会显著增强。

### 2.4 归档与规范（`doc材料/_archive/`）

早期版本的 **管理决策、SOP、Hive 资产 AI 化标准、Cursor 配置模板** 等，便于对照「演进过程」。若与当前团队最新规范冲突，以最新规范为准。

### 2.5 开发规则（`doc材料/开发规则/`）

- **`sql代码开发规则.mdc`**、**`sql格式化.mdc`**：SQL 编写与排版约定，可纳入 Cursor Rules 或团队文档。
- 其他 **Word** 等文件（若有）：公司/部门层面的数据开发规范原文，与 `.mdc` 配合使用。

## 三、`skills/` 目录

每个子目录下有一份 **`SKILL.md`**。将整个子目录复制到目标仓库的 **`.cursor/skills/<技能名>/`** 后，Cursor 可按 Skill 描述自动选用能力。

| 技能目录 | 典型用途 |
|----------|----------|
| **knowledge-base-builder** | 新项目或增量更新时，引导 AI 按步骤构建/更新知识库产物。 |
| **sql-review** | 对 Hive/Spark SQL 做静态审查、格式与质检。 |
| **dito-adhoc-query** | 临时取数、BRT 类需求、导出场景（结合历史经验路径时需在规则里配置实际目录）。 |
| **dito-troubleshooting** | 数据异常、指标对不上、任务空跑等问题排查思路与案例化协作方式。 |

复制后若路径与原作者机器不一致，请在本机 **`.cursor/rules`** 或 Skill 内说明中改为你的 `DDL/`、`知识库_*.json` 等实际位置。

---

## 四、`cursor对话样例/` 目录

Markdown 文件为 **对话实录类示范**（非可执行代码），用于学习：

- 如何分步骤让 AI 理解 DDL、血缘、取数约束；
- 如何描述需求、贴表名/分区、迭代 SQL。

可直接在 Cursor 中打开对应 `.md` 作为「提示词与流程」参考，**不要**把其中的内网路径、账号信息照搬到不可信环境。

---

## 五、`excel格式转换脚本/` 目录

- **`excel_to_json_gui_toon.py`**：从 Excel 导出/转换为便于 AI 消费的 JSON、TOON 等格式的 **脚本**。
- **`ExcelToTOON.exe`**（若存在）：图形界面或小工具封装；请在 **可信环境** 运行，并注意与 Python 版本、依赖一致。

用于将 **模型 Excel** 转为与 `模型_toon` 类似的文本结构，减轻「大表手工整理」成本。

---

## 六、推荐阅读顺序（首次接触）

1. 阅读 **`doc材料/README.md`** 与本文 **`使用说明.md`**。  
2. 浏览 **`doc材料/DITO/`** 下若干 **`.md`** 全景与速查，建立「有什么资产」的印象。  
3. 打开 **`通用知识库整理提示词.md`** 或 **`DITO/知识库构建提示词.md`**，理解如何驱动 AI。  
4. 将 **`skills/`** 中需要的技能拷入自己的 `.cursor/skills/`，按需配置规则路径。  
5. 参考 **`cursor对话样例/`** 中的对话方式，在自己的项目里试跑一轮简单问答或建表校验。
