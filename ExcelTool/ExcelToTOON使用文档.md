## 一、工具概述

ExcelToTOON 是一款桌面工具，用于将 SDM 模型 Excel 文件（`.xlsx` / `.xlsm`）批量转换为 TOON 格式文件。

TOON（Token-Oriented Object Notation）是一种为大语言模型优化的紧凑数据格式，相比 JSON 节省约 30%-50% 的 Token 消耗，适合作为 LLM 辅助数据开发的上下文输入。

### 核心能力

| 能力 | 说明 |
|------|------|
| 多Sheet批量转换 | 一次性处理数百个Sheet，逐一输出为独立的 `.toon` 文件 |
| 多种布局兼容 | 自动识别标准布局、数据抽取要求布局、无来源表布局等 |
| 关键字模糊匹配 | 自动匹配"源系统"/"来源系统"等变体写法，容错能力强 |
| 处理报告生成 | 输出 Markdown 格式处理报告，含成功/失败明细和堆栈信息 |
| 进度条实时反馈 | 大文件处理时显示进度和当前Sheet名称 |

## 二、环境要求

### 2.1 使用 exe 版本（推荐）

- **操作系统**: Windows 10 / Windows 11（64位）
- **无需安装** Python 或任何依赖，双击即可运行

### 2.2 使用 Python 源码

- Python 3.10+
- 依赖安装：

```
pip install pandas openpyxl
pip install tkinterdnd2  # 可选，提供拖放功能
```

- 启动方式：

```
python excel_to_json_gui_toon.py
```

## 三、操作流程

### 3.1 启动工具

双击 `ExcelToTOON.exe`，等待 3-5 秒（首次启动需解压运行时），工具窗口出现。

### 3.2 选择 Excel 文件

有两种方式选择文件：

- **点击"选择文件"按钮**：弹出文件选择对话框，选择 `.xlsx` 或 `.xlsm` 文件
- **拖放文件**（需安装 tkinterdnd2）：将 Excel 文件直接拖入窗口的拖放区域

选择文件后，工具自动解析所有 Sheet 名称并显示在复选框列表中。

### 3.3 选择要转换的 Sheet

- **全选**: 点击"全选"按钮，选中所有 Sheet
- **取消全选**: 点击"取消全选"按钮，清空选择
- **手动勾选**: 逐个勾选需要转换的 Sheet
- **筛选功能**: 在"筛选Sheet"输入框中输入关键词，支持空格分隔多个关键词进行联合筛选

> 筛选仅影响显示，不影响已勾选状态。例如：筛选后全选只选中当前可见的Sheet。

### 3.4 设置输出目录

- 默认输出目录为程序运行目录下的 `toon_output` 文件夹
- 可手动修改路径或点击"浏览"选择目标文件夹
- 如果目录不存在，工具会自动创建

### 3.5 开始转换

点击"开始转换"按钮：

1. **进度条**显示当前处理进度和正在处理的 Sheet 名称
2. 处理过程中按钮自动禁用，防止重复提交
3. 处理完成后弹出结果对话框，显示成功/失败数量

### 3.6 查看结果

转换完成后：

- **界面结果区**：显示处理汇总（成功数、失败数、耗时）及失败Sheet明细
- **输出目录**：
  - 每个 Sheet 生成一个 `{Sheet名}_{目标表英文名}.toon` 文件
  - 自动生成 `处理报告.md`，包含完整的成功/失败列表和错误堆栈信息

## 四、输入文件格式要求

### 4.1 支持的文件格式

| 格式 | 说明 |
|------|------|
| `.xlsx` | 标准 Excel 工作簿 |
| `.xlsm` | 启用宏的 Excel 工作簿 |

### 4.2 Sheet 布局要求

工具通过关键字自动识别 Sheet 结构，**必需关键字**和**可选关键字**如下：

| 关键字 | 是否必需 | 说明 | 可接受的变体写法 |
|--------|----------|------|-----------------|
| 目标表英文名 | **必需** | 用于确定输出文件名 | 目标英文表名、目标表名、英文表名 |
| 目标表中文名 | 可选 | 表的中文描述 | 目标中文表名、中文表名、目标中文名 |
| 主题域 | 可选 | 业务主题分类 | 主题、theme、数据分类 |
| 数据频度 | 可选 | 更新周期 | 数据频率、频度、更新频率 |
| 模型主键 | 可选 | 表的主键字段 | 主键、模型主键字段 |
| PDM字段英文名 | **必需** | 字段定义表头行 | PDM字段英文名称、字段英文名 |

> 如果 Sheet 中找不到"目标表英文名"（所有变体均不存在），该 Sheet 会被标记为失败，不生成 toon 文件。

### 4.3 支持的布局类型

#### 布局A：横向交替型（数据抽取要求型）

关键字与值在同一行交替排列：

```
主题域 | 专项分析 | 目标表英文名 | dm_xxx_d | 目标表中文名 | xxx表
数据频度 | D: Daily | 数据抽取要求 | I3级-Insert | 模型主键 | cell_key
```

#### 布局B：纵向表头型

关键字排列在表头行，值在下一行对应列：

```
主题域   | 目标表英文名   | 目标表中文名 | 数据频度 | 模型主键
PM       | dm_xxx_d      | xxx表       | day     | id
```

#### 布局C：无来源表型

没有来源系统/源表区域，直接从目标表信息跳到字段定义：

```
目标表信息...
详细字段信息
PDM字段英文名 | PDM字段类型 | PDM字段中文名（说明） | ...
field1       | string     | 字段说明              | ...
```

#### 布局D：投诉/多源型

来源表区域使用不同的列标题（如"源表描述"替代"来源系统"）：

```
源表关联信息描述
源表描述 | 源表英文名 | 表结构 | 是否主表 | 备注 | 任务描述 | 关联方式
dtsw    | table_a   | A     | 是      | ...  | ...     | ...
```

> 以上所有布局工具均可自动识别，无需用户干预。

### 4.4 来源表区域识别规则

来源表列表头行的识别依据为：在 PDM 字段行上方、含有以下任一关键字的**多列行**（排除只有一个单元格的节标题行）：

- 源表英文名 / 源表英文名称 / 源英文名
- 源系统 / 来源系统 / 源表描述 / 源表关联信息描述

### 4.5 不适用的 Sheet 类型

以下类型的 Sheet 无法被转换（会被标记为失败）：

- **目录页**：如"目录"、"导航底层"、"导航应用"等导航页面
- **元数据页**：如"修改记录"、"模板"等管理页面
- **参考数据页**：如纯数据映射表（没有标准SDM表结构）
- **GIS图层配置页**：如"gis图层"等非标准结构页面

## 五、输出格式说明

### 5.1 TOON 文件结构

每个 `.toon` 文件包含一个完整的表定义，结构如下：

```
theme_domain: 人口
table_name: dim_population
table_name_chinese: 2015和2020年普查人口
data_frequency: 维表
primary_key: 数据描述
source_table_name[0]:
fields[11]{PDM字段英文名, PDM字段类型, PDM字段中文名（说明）, ...}:
  area_name, string, 区域名称, null, null, null, ...
  province_code, string, 省编码, null, null, null, ...
```

#### 格式规则

| 规则 | 说明 |
|------|------|
| `key: value` | 简单键值对 |
| `key[N]{col1, col2, ...}:` | 结构化数组，N为元素数量，花括号内为列名 |
| `  val1, val2, ...` | 数组元素，缩进2空格，逗号分隔 |
| `null` | 空值 |
| `"带,特殊:字符"` | 含特殊字符的值用双引号包裹，内部 `"` 转义为 `\"` |

### 5.2 文件命名规则

输出文件名格式：`{Sheet名}_{目标表英文名}.toon`

- 文件名中的特殊字符（`\ / : * ? " < > |`）会被替换为下划线
- 示例：Sheet名 `F.28`，目标表 `dim_population` → 文件名 `F.28_dim_population.toon`

### 5.3 处理报告

每次转换自动在输出目录生成 `处理报告.md`，包含：

- **处理汇总**：成功数、失败数、合计、成功率、处理耗时
- **失败明细表**：失败的Sheet名称和失败原因
- **详细堆栈信息**：每个失败Sheet的完整Python堆栈跟踪（便于排查问题）
- **成功列表**：成功Sheet的名称、目标表名、输出文件名、字段数

## 六、常见问题

### Q1：启动 exe 后没有反应？

exe 文件首次启动需要解压内置的 Python 运行时，通常需要 3-5 秒。如果长时间无反应，检查：
- 是否被杀毒软件拦截（PyInstaller 打包的 exe 可能触发误报）
- 尝试右键"以管理员身份运行"

### Q2：大文件处理很慢？

工具使用 `pd.ExcelFile` 一次性加载文件对象，然后逐 Sheet 解析，已针对大文件优化。参考处理速度：
- 478个Sheet / 10MB文件：约 20 秒
- 847个Sheet / 12MB文件：约 27 秒

### Q3：Sheet 转换失败显示"未找到目标表英文名"？

说明该 Sheet 不是标准的数据表定义页。常见于目录页、修改记录页、导航页等。这些 Sheet 可以在转换前取消勾选。

### Q4：模糊匹配识别错误怎么办？

工具的模糊匹配阈值为 0.6（60%相似度）。如果出现误匹配，可以：
1. 检查 Excel 中的关键字拼写是否规范
2. 在源码 `KEYWORD_ALIASES` 字典中添加新的别名

### Q5：输出的 toon 文件中有 `nan` 列名？

部分 Excel 的列标题行在末尾有空列，pandas 读取后列名为 `nan`。这不影响实际数据的准确性，后续使用时可忽略该列。
